lcof03 数组中的重复数字: done not best

在一个长度为 n 的数组 nums 里的所有数字都在 0～n-1 的范围内。数组中某些数字是重复的，但不知道有几个数字重复了，也不知道每个数字重复了几次。请找出数组中任意一个重复的数字。

  1. 想法1：排序遍历 O(nlogn) + O(1)
  2. 想法2：哈希表遍历 O(n) + O(n)
  3. 想法3：不可能有更优的了，我忽略了题目中的所有数字都在 0～n-1 的范围内。

最优解: 原地置换。若无重复数字，排序后数字i的下标为i。故遍历nums，若nums[i] = n != i，则swap(nums[i], nums[m])，交换过程中遇到重复数字即可返回。

精选题解

所谓原地交换，就是如果发现这个坑里面的萝卜不是这个坑应该有的萝卜，就看看你是哪家的萝卜，然后把你送到哪家，再把你家里现在那个萝卜拿回来。

拿回来之后，再看看拿回来的这个萝卜应该是属于哪家的，再跟哪家交换。

就这样交换萝卜，直到想把这个萝卜送回它家的时候，发现人家家里已经有了这个萝卜了（双胞胎啊），那这个萝卜就多余了，就把这个萝卜上交给国家。

2/25 pday867 & lcof04

pday867 转置矩阵: done best
lcof04 二维数组中的查找: done best

在一个 n * m 的二维数组中，每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个高效的函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。

  1. 想法1：这题我做过，可以O(n + m)

最优解: 线性查找。从右上角开始，大了就向左，小了就向下。

精选题解

类似于二叉排序树，左上都是小的，右下都是大的。
